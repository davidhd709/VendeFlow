## Resumen rápido de cobertura observada

- Backend con mejor cobertura: `auth`, `leads`, `sales`, `analytics`, `tasks`, `campaigns`, `website-builder` + `tenant-isolation` integración.
- Frontend con cobertura activa en módulos públicos críticos (sitio, catálogo, detalle, cotización, resolución de tenant) y guardias base.
- Estado suite frontend actual: **18 suites / 59 tests pasando**.
- Módulos aún con deuda de tests: `companies`, `offices`, `users`, `goals`, `templates`, `website-config`, `files`, `seller assignments`, `comments`, `follow-ups`.

## Riesgos multitenant/seguridad detectados (sin corrección en esta tarea)

1. `files`: falta de tests automáticos de scoping cloudinary por tenant y validaciones de rechazo.
2. `website-builder` público: ya consumido en frontend, pero aún sin cobertura E2E completa de publicación.
3. `audit`: sin tests directos de sanitización de metadata (riesgo de fuga accidental de PII en evolución futura).
4. Múltiples módulos sin pruebas de regresión cross-tenant específicas (aunque existe test de integración base).

## Observaciones visuales (frontend)

- Dashboards y web pública funcionales, pero con deuda de polish consistente en jerarquía visual y experiencia responsive entre módulos.
- La ruta pública `/catalogo/:slug` ya está implementada; pendiente siguiente: stock real por oficina en API pública y SEO/copy avanzado.
